Tak oto wyglda wykres, ktry bdzie przeze mnie poprawiany:

W oryginale jest on rwnie interaktywny, o czym mona si przekona, klikajc w pierwszy link. Po najechaniu kursorem myszki na dany region, wywietla si informacja o nazwie prowincji, liczbie potwierdzonych przypadkw i przypadkw aktywnych.

Co naley poprawi?

Zauwaam liczne bdy w wykonaniu tego wykresu:

  • Na pierwszy rzut oka ra mnie umieszczone w zupenie losowych miejscach na wykresie liczby. Nie wiadomo ani jak zmienn one przedstawiaj, ani, do ktrego regionu si odnosz. W niektrych przypadkach bowiem jedna liczba jest zapisana na granicy trzech regionw.
  • Skala jest ciga, co raczej nie jest dobrym wyborem dla tego typu wykresu, gdy ludzkie oko ma problem z ocen subtelnych rnic midzy odcieniami kolorw.
  • Wybrane odwzorowanie (Merkatora) nie jest dobrym wyborem dla Kanady - kraju o duej rozcigoci rwnolenikowej pooonego stosunkowo blisko bieguna pnocnego.
  • Oznaczona przez te koo zmienna 'Active Cases' zlewa si w niektrych przypadkach z kolorem ta. W dodatku nigdzie na wykresie nie ma zdefiniowanej dla niej skali.
  • Biorc pod uwag rwnie inne wykresy na tym dashboardzie, trudno tak naprawd okreli, ktry dokadnie dzie zosta przedstawiony na kartogramie.
  • Na tego typu wykresach nie jest rekomendowane stosowanie danych bezwzgldnych, lepiej przedstawi dane wzgldne, uwzgldniajce populacj poszczeglnych regionw.

Kod niezbdny do wygenerowania poprawnej wizualizacji

Poprawy wykresu zdecydowaem si dokona na podstawie oficjalnych danych kanadyjskiego rzdu: https://health-infobase.canada.ca/covid-19/#a4

#categorizing the number of cases and assign each category to each row
def set_cat(row):
    if row['TotalCasesPerPopulation'] == 0:
        return '0'
    if row['TotalCasesPerPopulation'] > 0 and row['TotalCasesPerPopulation'] < 101:
        return '1 - 100'
    if row['TotalCasesPerPopulation'] > 101 and row['TotalCasesPerPopulation'] < 201:
        return '101 - 200'
    if row['TotalCasesPerPopulation'] > 201 and row['TotalCasesPerPopulation'] < 301:
        return '201 - 300'
    if row['TotalCasesPerPopulation'] > 301 and row['TotalCasesPerPopulation'] < 401:
        return '301 - 400'
    if row['TotalCasesPerPopulation'] > 401 and row['TotalCasesPerPopulation'] < 501:
        return '401 - 500'
    if row['TotalCasesPerPopulation'] > 501:
        return '501 and higher'

Co si zmienio?

  1. Podjem decyzj o pozbyciu si zbdnych liczb z kartogramu. Gwny przekaz dotyczcy liczby zachorowa na sto tysicy mieszkacw jest przekazany przez kolory, nie ma sensu dublowa tej informacji.
  2. Podzieliem cig zmienn w dyskretne przedziay, dziki temu czytelnik powinien atwiej zauway rnice.
  3. Zamiast danych bezwzgldnych, wyliczyem wzgldny wskanik zaleny od liczby mieszkacw danego regionu, dziki czemu ten wykres przedstawia nam realn sytuacj w poszczeglnych regionach.
  4. Zamiast odwzorowania Merkatora, zdecydowaem si wybra odwzorowanie stokowe - oficjalne odwzorowanie Kanady.
  5. Po najechaniu kursorem myszy na wykres uytkownik otrzyma peen zestaw informacji, zawierajcy dane wzgldne i bezwzgldne dotyczce cznej liczby przypadkw, ale take tych najnowszych z perspektywy danej daty.
  6. Wykres jest animowany, po naciniciu przycisku "play" zaczyna porusza nam si o czasowa i wykres jest odpowiednio modyfikowany.
